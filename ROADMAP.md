# Brand Kit Studio Roadmap

Brand Kit Studio is being developed as a local-first implementation tool for brand systems, not as a hosted design suite.

## Current Focus

1. Make imported project context richer so brand outputs are based on better raw material.
2. Reduce manual steps between choosing a direction and getting a usable installed kit.
3. Keep the trust boundary explicit around local capture, provider use, and install writes.

## Near-Term Priorities

1. Richer file and screenshot extraction
   Improve what the app can pull from uploaded files and screenshots before direction generation.
2. `DESIGN.md` import and parsing
   Let existing design-system documents pre-fill the studio so repeat projects do less manual work.
3. Faster direction-to-logo workflow
   Add optional autopilot steps after a redesign direction is selected while preserving approval gates.

## Next Wave

1. AI-assisted brand direction generation behind explicit provider settings.
2. Better install preview and merge reporting for project files.
3. More reliable local preview capture and applied-frame rendering.

## Later

1. Richer raster social and banner exports after logo approval.
2. Optional public website capture only behind an approval-gated safety step.
3. Broader portfolio-level brand governance features once the single-project loop is stronger.

## Not The Goal Right Now

- Unreviewed public web scanning
- A multi-user cloud design platform
- Hidden provider calls without explicit local configuration
