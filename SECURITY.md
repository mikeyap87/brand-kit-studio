# Security Policy

## Reporting A Vulnerability

Please do not open a public issue for security problems that could expose:

- API keys or local environment handling
- private workspace paths or project data
- unintended file-write behavior
- localhost capture or provider-bridge risks

Instead, report the issue privately to the maintainer with:

- a short description
- affected files or workflow
- reproduction steps
- impact assessment

## Supported Scope

Security review is currently focused on the active mainline code in this repository. Older local experiments or unpublished branches may not receive the same support.
